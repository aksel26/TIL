# 모두의 약수

### 내 코드 

```c
#include <stdio.h>

using namespace std;

int main()
{
    int n, cnt = 0;
    scanf("%d", &n);

    for (int i = 1; i <= n; i++)
    {
        for (int j = 1; j <= i; j++)
        {
            if (i % j == 0)
            {
                cnt++;
            }
        }
        printf("%d ", cnt);
        cnt = 0;
    }

    return 0;
}
```

- 이렇게 하면 `O(n^2)` 시간 복잡도가 나온다.



<br/> 

### 참고

```c
#include <stdio.h>

using namespace std;
int cnt[50001];
int main()
{
    int n;

    scanf("%d", &n);

    for (int i = 1; i <= n; i++)
    {
        // i 의 배수
            // j 증가 연산 수가 배수만 증가하면 되니까 복잡도가 줄어드는 효과가 있다.
        for (int j = 1; j < n; j = j + i)
        {
            cnt[j]++;
        }
    }

    for (int i = 1; i < n; i++)
    {
        printf("%d ", cnt[i]);
    }

    return 0;
}
```

2중 for문이어도 `j = j + i` (i의 배수) 를 이용하면 `O(nlogn)` 까지는 아니더라도  시간복잡도가 현저히 줄어들 수 있다.
